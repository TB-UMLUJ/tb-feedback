
دليل رفع المشروع على GitHub وربطه بـ Vercel (خطوة بخطوة)
=========================================================

هذا الدليل يشرح لك كيفية رفع ملفات مشروعك الحالية إلى مستودع GitHub، ثم ربط هذا المستودع بمنصة Vercel للحصول على رابط ويب مباشر للتطبيق.

المتطلبات المسبقة:
------------------
1. حساب على GitHub (github.com).
2. حساب على Vercel (vercel.com).
3. برنامج Git مثبت على جهازك (git-scm.com).
4. بيانات Supabase الخاصة بك (الرابط والمفتاح) التي استخدمتها في الكود.


القسم الأول: رفع المشروع إلى GitHub
-----------------------------------

1. تجهيز المشروع:
   - افتح مجلد مشروعك على جهازك.
   - تأكد من وجود ملف باسم `.gitignore` (إذا لم يكن موجوداً، أنشئه وضع فيه السطرين التاليين):
     node_modules
     .env
     dist
     .DS_Store

2. إنشاء مستودع جديد (Repository):
   - اذهب إلى github.com وسجل الدخول.
   - اضغط على علامة (+) في الزاوية العلوية واختر "New repository".
   - سمّه مثلاً: `amalj-maintenance`.
   - اجعله "Public" أو "Private" (كما تحب).
   - اضغط "Create repository".

3. رفع الملفات (عبر موجه الأوامر/Terminal):
   - افتح الـ Terminal أو CMD داخل مجلد مشروعك.
   - اكتب الأوامر التالية بالترتيب (اضغط Enter بعد كل سطر):

     git init
     git add .
     git commit -m "Initial commit - Complete App"
     git branch -M main
     
     (السطر التالي انسخه من صفحة المستودع التي أنشأتها في GitHub)
     git remote add origin https://github.com/TB-UMLUJ/tb-feedback.git
     
     git push -u origin main

   - الآن، إذا ذهبت لصفحة GitHub وحدثتها، ستجد ملفاتك قد ظهرت هناك.


القسم الثاني: النشر عبر Vercel
------------------------------

1. ربط Vercel بـ GitHub:
   - اذهب إلى vercel.com وسجل الدخول (يفضل باستخدام حساب GitHub).
   - في لوحة التحكم (Dashboard)، اضغط على زر "Add New..." ثم اختر "Project".
   - ستظهر لك قائمة بمستودعاتك في GitHub. ابحث عن `amalj-maintenance` واضغط زر "Import" بجانبه.

2. إعداد المشروع (Configure Project):
   - في صفحة الإعداد، لا تغير "Framework Preset" (سيتعرف عليه تلقائياً كـ Vite).
   - اذهب إلى قسم **Environment Variables** (المتغيرات البيئية) وهذا مهم جداً لعمل قاعدة البيانات.
   - أضف المتغيرات التالية (انسخ القيم من ملف مشروعك أو من Supabase مباشرة):

     الاسم (NAME): VITE_SUPABASE_URL
     القيمة (VALUE): رابط مشروعك (مثال: https://xyz.supabase.co)
     -> اضغط Add

     الاسم (NAME): VITE_SUPABASE_ANON_KEY
     القيمة (VALUE): مفتاح anon الطويل
     -> اضغط Add

3. النشر (Deploy):
   - اضغط على زر **Deploy**.
   - انتظر دقيقة أو دقيقتين حتى ينتهي البناء (Build).
   - ستظهر شاشة احتفال، اضغط على صورة الموقع أو زر "Continue to Dashboard".
   - اضغط على زر "Visit" لرؤية موقعك.

القسم الثالث: ملاحظات هامة
--------------------------

1. التحديثات:
   - في المستقبل، إذا عدلت أي كود في جهازك، فقط اكتب هذه الأوامر لرفع التعديل، وسيقوم Vercel بتحديث الموقع تلقائياً:
     git add .
     git commit -m "وصف التعديل"
     git push

2. قاعدة البيانات:
   - موقعك الآن يعمل على الإنترنت ومتصل بنفس قاعدة بيانات Supabase. لا تحتاج لتغيير أي شيء في Supabase.

3. جعل الموقع تطبيق (PWA - اختياري):
   - الرابط الذي حصلت عليه من Vercel يمكن فتحه في جوال الآيفون (Safari) واختيار "إضافة إلى الشاشة الرئيسية" (Add to Home Screen) وسيظهر كأنه تطبيق تماماً.
